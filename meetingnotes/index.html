How it works

It listens to your system audio + mic — so it works the same for Meet and Zoom, with no bot or vendor API.

1
Capture localRecords your microphone (“Me”) and your computer’s audio output via WASAPI loopback (“Others”) to two WAV files.
2
Live draft CPUA small faster-whisper model transcribes each window in near-real-time and streams it to the dashboard.
3
Batch pass GPU or CPUOn stop: large-v3 transcription + speaker diarization + an LLM produce the polished transcript, summary, and action items. Offloads to a SLURM GPU node if configured.
4
Review & ask dashboardBrowse meetings, rename speakers, tick off action items, search across everything, and chat with the transcript.

Quickstart

Python 3.10–3.12. faster-whisper needs no GPU; everything runs on CPU (just slower).

# install
python -m venv .venv && .venv\Scripts\activate    # Windows
pip install -e .

# configure (optional: HF token only for the gated pyannote diarizer)
copy .env.example .env

# run the dashboard
meetingscribe serve            # → http://127.0.0.1:8765

# …or record from the terminal
meetingscribe record -t "Team standup" -p meet

Then join your Meet/Zoom call, hit Start, and hit Stop when you’re done. Notes appear automatically.

Honest by default

A few things we’re upfront about.

Auto-record, not auto-join

It records the meetings you attend (hands-free, on schedule). A bot that joins a call while you’re away would need the Zoom Meeting SDK — deliberately out of scope for a key-free tool.

Diarization quality

The default diarizer is fully key-free. For best-in-class speaker separation, enable the optional pyannote backend (one free model-terms acceptance on Hugging Face).

You own the data

Recordings, transcripts, and notes live in a folder on your disk. Delete a meeting and it’s gone — no server copy.

🔒 No analytics, no phone-home, no account. The only network calls are optional: downloading model weights once, and (if you opt in) a remote GPU you control.
